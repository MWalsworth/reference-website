Naming convention for all filenames, paths and folders

Proper naming conventions for all filenames, paths and folders are as follows.

  • There must be no spaces
  • must be all lowercase
  • and contain only letters, numbers and dashes (-)

Best practices for commit messages

The best practices when commiting a message are:

  • Minimum 3 words & 10 characters
  • Proper spelling & grammar
  • Capital starting letter, no period
  • Must start with an imperative verb: Create, Fix, Add, Solve, etc.

Weeks 4 & 5

Structuring content:

Header

The header is the introductory content at the top of the page. This is where you would find logos and or navs. You may also find it within sections and articles to introduce the content for headings, subheadings and authors.

Footer

The footer is the botom of the webpage. It will include links, copyright informationsm extra information about the company, careers and terms and services.

Main

The main element wraps around the main content of the page. It's used only once per page.

Section

Sections are used to wrap around related pieces of content. Each section typically have their own heading.

Article

This is self-contained content in a web page. It should be distributable and reusable because of it being independent.

Aside

The aside element is uusally seen as a sidebar on a website. It represents a portion of a document that is indirectly related to the documents main content.

A Nav Tag

The nav provides links either within the current webpage or to others.

Comments

The tag to make a comment is < ! -- ... --> A comment is used to explain your code, or insertion points at a later date. They are not viewed in the browser.

Generic elements

Generic elements are used to target content which is most often used later to manipulate styling with CSS (or later with Java script)

< div > are block elements used to group elements together

< span > are an inline container used to mark up a part of a text, or document.

Attributes

Attributes for HTML provide additional information about the HTML elements.

For example: href="" is an attribute

Linking to an absolute url/path

Absolute URLS or Absolutely Paths link websites to other resources on the internet

Linking to a relative url/path

This is a relative url or relative path. It links the path to the current page because it assumes it has the the same protocol, subdomain and domain as the page its on.

Linking to an element on the same page

Linking to an element on the same page is a little different, you need to create an anchor for the link. To do this you must give the appropriate element an id="Content" attribute.

Understanding nesting (parent/child relationship)

When learning HTML, the parent-child relationships of the elements is critical to good structure & code understanding. A parent is an element that is directly above and connected to an element in the document tree. An element that is directly contained by another element is said to be the child of that element.

Importance of indentation

When writing HTML it is important to properly format your code using good indentation. You can indent elements by moving them two spaces to the right. This will make your code more readable by other developers and shows the relationship between the child and parent HTML elements. (Wilkins, Jessica 2021)

Week 6 - Accessibility

The importance of building accessible websites

It's important to build a website that has the abilitiy to be used by everyone, hence accessibility. This includes users with disabilities. The idea is so that everyone can have the same ideal experience and be able to access the information equally.

Web Content Accessibility Guidelines 2.0 (WCAG)

Websites must comply with WCAG2.0 in order to be considered accessible. And therfore broken into four main categories called POUR.

    P.O.U.R stands for:
  • Perceivable
  • Operable
  • Understandable
  • Robust

Perceivablethat all information must be presented in such a way to each user despite which senses they have use of, or lack thereof.

Operable means that each user must be able to locate, navigate and interact with webpages in a myriad of ways

Understandable is just what it implies. All content must be understandable by its users.

Finally, Robust means that websites must adopt web-standards so that they will work across all browsers and assistive technologies.

Types of disabilities / impairments users may have

It's important to consider all types of disablities/impairements when creating websites in order to make your website accessible to all. Such disabilities/impairements include:

  • Visual Impairements
  • Cognitive Impairements
  • Auditory Impairements
  • Mobility/dexterity Impairements

Other helpful tips to consider when making your website accessible is to:

  • Semantic HTML provide extra information that help people use our website.
  • Keep your code & your content in your HTML documents in the order you expect them to be read
  • Break up your content using heading tags. They're nice visual breaks, but screen readers use them as jumping points.

Visual impairements include: Blindness; partial or full, colour blindness, cataracts, glaucoma, age, etc..

  • Allow text to be resized
  • Have good contrast in colours
  • Test for colour blindness related issues
  • Website must work well with screen readers
  • Use proper alt attributes

Cognitive impairements include: Dyslexia, memory issues, problem solving issues, attention deficits, hyperactivity, reading abilities, etc.

  • Make it clear where in the website the user is
  • Organize your content correctly
  • Make the text easily scannable
  • Clearly mark links
  • Use plain, simple language

Auditory impairements inlcude: Deafness, partial deafness, etc.

  • Provide text captions and subtitles - when including audio or video on your website.
  • Users may need closed caption while viewing videos due to language barriers or being in a noisy room.

Mobility impairements include: Missing a hand, limited movement, difficulty with fine control, trouble holding a mouse, tremors or shakes, etc.

  • Recognize and prepare for the fact that not everybody will use a mouse
  • Make the website keyboard accessible by making sure you are giving your users alternate ways to access your information
  • hit areas of links and buttons

WAI-ARIA

Web Accessibility Initiative (WAI) has a specification called Accessible Rich Internet Application (ARIA). ARIA helps makes web content and applications more accessible to people with disabilities or impairements by defining ways for functionality to be provided to assistive technology like screen readers, screen magnifiers, switch controls, closed captioning etc.

Landmark roles

ARIA has a set of landmark roles to help those using screen readers to jump directly to specific sections within websites. This includes roles such as:

  • header role="banner" - this is added in the header tag for defining an element as the masthead.
  • nav role="navigation" - this is added to the top to define the elements as the primary navigation.
  • main role="main" - this is added to the main to define the element as the main content of the page.
  • aside role="complimentary" - this is added to an aside tag as being additional content related to the primary content.
  • footer role="contentinfo" - this is usually added to the footer to define an element as information about the content like copy right, terms, privacy policy, careers ect.

Labeling links

This semester we have using the < aria - label="" > attribute tags to create labels in fields that have no visual label. Screen readers would give that voice to the missing visual component so that those with these impairements are not losing out on their experience when interacting with our website. Note: we use the tag < role="presentation" > for screen readers to skip these labels when images have no impact on experience.

Image descriptions

It is imperative descriptions are added to images with an alt atribute. Thse have two purposes:

  • To describe a photo if it doesn't download, or,
  • To describe the photo via screen reader to users with visual impairemnts.

Note: As mentioned before, you can apply role="presentation" if the image is purely decorative.

Skip links

These are placed at the top of the page and offer users a way to jump to specific areas of the page as you would have experienced when interacting with this page.

Introduction to CSS

In Weeks 8 through to the end of the semester we have been working on CSS.

CSS stands for Cascading Style Sheets is the language we use to control the appearance of our HTML pages.

CSS files mist be linked in the index.html so the browser knows where to look for style sheet. It must be attached within the head of the html document using the link element.

As indicated above, the link element must have two attributes in order to be attach the CSS to the HTML. These are:

  • href - this is to used to point to the location of the css file in whichever folder it may be
  • rel - this is to define that this item is a style sheet

It is important to note, that like everything CSS has proper syntax which is written as follows. h1 { color:black; }

As you can see, it is composed of a:

  • Rule set - this is contains the selector and one or plu declarations surrounded by curly braces ({})
  • Selector - this is the identifer that the HTML element should be targeting
  • Declaration - assigns calue to a property, seperated by a colon. Each declaration must end with a semi-colon.
  • Property - this is the type of design you want to add. For example: width, color, border, etc.
  • Value - accepted value for the property. For example: blue when property is related to the the background-color

Font properties

Font-Size

For adjusting the size of the font: a single number with a unit (rem [the best choice], em, px)
Ex: 2rem, 1.125em, 24px

Font-Weight

Sets how thick or thin characters in text should be displayed
Ex: bold, normal, 700

Font-Style

Specifies font style for text
Ex: italic or normal

Line-Height

For adjusting the line-height: a single number, a multiplier that multiplies the font-size
Ex: 1.5

Font-Family

The font-family specifies the font for an element that is oterhwise not specified. If the brower support of the first font it will try the second. This is because it relies on a fall back system.

  • A Family-name - is the name of a font family.
  • A Gerenic family - is the name of a generic family like serif font athlelas,

Text decorations

Text-Transform

Change capitalization such as uppercase, lowercase, capitalize. It's a good idea to use CSS to make all caps instead of writing it directly in HTML for accessibility reasons.
Ex: uppercase, lowercase, capitalize

Text-Decoration

Specifies the decoration added to text. The only things in a website that should be underlined are links.
Ex: none, underline, line-through, overline

Text-Align

Specifies the horizontal alignment of text in an element
Ex: left, center (American spelling), right, justify

Text-Indent

Indent the first line of text like traditionally printed materials
Ex: 1rem, 2em, 20px, 25%

Text-Shadow

Adding a shadow to the text. Specified like this: text-shadow: horizontal-offset vertical-offset blur-radius colour
text-shadow: 2px 2px #ff0000;

Styling Links

Styling links in websites is not really different from other elements, but they must be styled directly, and do not inherit certain properties from their parent elements.

Links have a bunch of states that we can also style when a user interacts with them.

  • :link — for styling the default link state
  • :visited — for styling the link after it']s been visited by the user
  • :hover — for styling the link when the mouse is sitting over it
  • :focus — for styling the link when the keyboard has tab-focused it
  • :active — for styling the link when the mouse button is pressed down on it

It's important that these link states go in this direction: link, visited, hover, focus, active. There's a fun a mnemonic to remember the order: “Lord Vader's Handle Formerly Anakin”.

Everything is a Box Model

it is easiest to think as HTML is made up of a box. That measns that HTML is wrapped around by whats called the box model which contains four parts:

  • Content - Content is the first layer which is definied as the basic box
  • Padding - Padding is the area surrounding the content. It what gives content breathing room by pushing the esge of the box away from the content.
  • Border - The stroke surrounding padding is called a border. If you had a background color it would stop here.
  • Margin - is area outside of the box. It is a transparent layet pushing away other boxes.

Here is a reference to help better understand all the parts listed above.

Having a basic idea of the box model is what will help to better understand padding and margin. In a nutshell, padding and margin are properties used to apply space either inside the box (padding) or outside the box(margin)

Padding

To further explain, Padding adds space inside the box by pushing the content away from the porder. This would mean that any background color is visible with the padding.

Margin

To further explain, the margin is the invisible space around your box, it adds space surounding your box by pushing away other content away from it.Note: Margins can have positive or negative values

Shorthand

Margin and padding can be specified with their individual properties, but can also be written shorthand as to save time on tediousness.

It is very important that you always start clocklwise around the box, therefore always starting with the top.

It is helpful that the browser will match the opposing sides valies if a number is missing.

  • If there is three numbers provided it will be applied as top, left/right, bottom
  • If there are only two numbers provided it will be applied top/bottom & left/right
  • If there is only one number it will be specify that the margin or padding is equivalent on all our sides.
